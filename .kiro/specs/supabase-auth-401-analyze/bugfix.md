# Bugfix Requirements Document

## Introduction

При загрузке файла для анализа пользователь успешно загружает файл через `/api/upload`, но при попытке запустить анализ через `/api/analyze` возвращается ошибка 401 (Unauthorized). Проблема заключается в том, что сессия Supabase не передаётся корректно между фронтенд-запросом и API роутом, или сессия истекает между загрузкой файла и запуском анализа.

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN фронтенд отправляет POST запрос к `/api/analyze` с `credentials: 'include'` THEN система возвращает ошибку 401 с сообщением "Требуется авторизация"

1.2 WHEN `createRouteHandlerClient({ cookies })` вызывается в `/api/analyze/route.ts` THEN `supabase.auth.getSession()` возвращает `null` несмотря на то, что пользователь авторизован

1.3 WHEN пользователь успешно загружает файл через `/api/upload` (сессия работает) THEN при переходе на страницу результата и запуске анализа сессия теряется

### Expected Behavior (Correct)

2.1 WHEN фронтенд отправляет POST запрос к `/api/analyze` с `credentials: 'include'` THEN система должна успешно получить сессию пользователя и начать анализ

2.2 WHEN `createRouteHandlerClient({ cookies })` вызывается в `/api/analyze/route.ts` THEN `supabase.auth.getSession()` должна вернуть валидную сессию пользователя

2.3 WHEN пользователь переходит на страницу результата после загрузки файла THEN сессия должна оставаться валидной для запроса к `/api/analyze`

### Unchanged Behavior (Regression Prevention)

3.1 WHEN пользователь загружает файл через `/api/upload` THEN система должна продолжать успешно проверять сессию и создавать запись в базе данных

3.2 WHEN пользователь не авторизован и пытается загрузить файл THEN система должна продолжать возвращать ошибку 401

3.3 WHEN анализ успешно запущен и завершён THEN система должна продолжать корректно обновлять статус и сохранять результаты в базе данных
