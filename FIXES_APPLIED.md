# Применённые исправления для ошибки анализа

## Что было исправлено

### 1. Аутентификация в `/api/analyze` ✅

**Проблема:** Функция `createRouteHandlerClient` не импортировалась, что приводило к ошибке при получении сессии.

**Решение:**
- ✅ Добавлены импорты: `createRouteHandlerClient` и `cookies` из `next/headers`
- ✅ Реализована двухуровневая аутентификация:
  1. **Первичный метод** — получение сессии из cookies (для браузерных запросов)
  2. **Fallback метод** — JWT токен из Authorization header (для API запросов)
- ✅ Добавлено подробное логирование для отладки

### 2. Обработка ошибок AI API ✅

**Проблема:** Ошибки от AI API не логировались полностью, что затрудняло отладку.

**Решение:**
- ✅ Добавлено логирование для каждого AI API (Gemini, Cohere, Qwen)
- ✅ Реализована цепочка fallback:
  1. Пытаемся Gemini
  2. Если ошибка → пытаемся Cohere
  3. Если ошибка → пытаемся Qwen
- ✅ Добавлено логирование stack trace для ошибок

### 3. Переменные окружения ✅

**Проблема:** `.env.example` не содержал все необходимые переменные для AI API.

**Решение:**
- ✅ Обновлен `.env.example` с поддержкой всех AI API:
  - `QWEN_API_KEY` (рекомендуется)
  - `GEMINI_API_KEY` (опционально)
  - `COHERE_API_KEY` (опционально)

## Что нужно сделать в Vercel

### Шаг 1: Проверить переменные окружения

1. Открой **Vercel Dashboard → Settings → Environment Variables**
2. Убедись, что установлены:
   - ✅ `NEXT_PUBLIC_SUPABASE_URL`
   - ✅ `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - ✅ `SUPABASE_SERVICE_ROLE_KEY` (Secret key `sb_secret_...`)
   - ✅ Хотя бы один AI API ключ:
     - `QWEN_API_KEY` (рекомендуется)
     - или `GEMINI_API_KEY`
     - или `COHERE_API_KEY`

### Шаг 2: Перезагрузить приложение

1. Открой **Vercel Dashboard → Deployments**
2. Нажми на последний деплой
3. Нажми кнопку **Redeploy** (или просто сделай новый git push)

### Шаг 3: Проверить логи

1. После перезагрузки открой **Vercel Dashboard → Deployments → [последний деплой] → Logs**
2. Загрузи файл для анализа
3. Посмотри на логи — ищи строки с `[POST]`:
   - `[POST] Got sessionUserId from cookies: ...` — сессия получена ✅
   - `[POST] Trying Gemini...` — пытаемся Gemini
   - `[POST] Gemini success` — Gemini сработал ✅
   - Если ошибка — посмотри на сообщение об ошибке

## Если всё ещё не работает

### Проверка 1: RLS политики

Если видишь ошибку про RLS или UPDATE:

1. Открой **Supabase Dashboard → SQL Editor**
2. Создай новый query
3. Скопируй и выполни SQL из `supabase/migrations/20260311_allow_update_analyses.sql`

### Проверка 2: API ключи

Если видишь ошибку типа "API error: 401":

1. Проверь, что API ключ скопирован правильно (без пробелов)
2. Убедись, что ключ активен в соответствующем сервисе
3. Если ключ неверный — удали его из Vercel и используй другой AI API

### Проверка 3: Логирование

Если не видишь логов в Vercel:

1. Убедись, что ты смотришь на правильный деплой (последний)
2. Убедись, что ты смотришь на POST запрос к `/api/analyze`
3. Попробуй перезагрузить страницу логов (F5)
